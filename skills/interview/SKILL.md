# Interview

Review the conversation history and relevant repository context before asking anything the available evidence can answer.

Ask one focused question at a time. Use the platform's structured question interface when it is available and appropriate; otherwise ask directly. Prefer concrete choices when they make the tradeoff easier to evaluate, while always allowing the user to supply a different answer.

Continue with follow-up questions until the objective, scope, constraints, priorities, and success criteria are clear enough to act without material guesswork. Do not repeat questions the user has already answered.

When the interview is complete, summarize the agreed direction and either continue the requested task or wait for explicit approval when the next step would create a meaningful external effect.
